Tessera Open Source Environment for Deep Analysis of Large Complex Data

The Power D&R and R

Tessera implements the Divide & Recombine statistical approach for deep analysis of larger and more complex data that leads to simple parallel computation. You program in the elegantly designed, powerful R language front end, and can apply the 1000s of methods of statistics, machine learning, and visualization in R.

The Tessera back end is a parallel computing environment doing D&R computations managed for you by Tessera. Whether you compute on a workstation, a small cluster, or a very high-end cluster, Tessera can greatly increase the size and complexity of data you are able to analyze. Getting started is easy.

Tessera Components

Divide and Recombine (D&R)

A statistical division method divides the data into subsets that are stored in objects with the same data structure. Each analytic method of the analysis is applied to the subsets independently without communication among the computations, which makes them embarrassingly parallel, the simplest parallel computation. A statistical recombination method is applied to the outputs of each analytic method, also stored with the same data structure, to form the final D&R result. The recombination can have a significant component of embarrassingly parallel computation. All of this makes D&R computation very simple, fast, and readily able to exploit parallel computational environments, especially distributed ones like Hadoop running on a cluster. Statistical accuracy of D&R results is important too. It depends strongly on the division and recombination methods. D&R research in statistical theory seeks methods that lead to high accuracy. This research replaces having to develop parallel algorithms for the 1000s of methods of statistics, machine learning, and visualization. See our publications below, some of which address D&R statistics.

Tessera with the Hadoop Back End

Tessera is powered by D&R. R was an obvious choice for a front end. We will stick with R until something better comes along. But we have a wide choice for back ends to carry out the parallel computation. Our first choice, with more options to follow, was Hadoop because it is readily available from many sources and widely supported. Hadoop distributes D&R subsets across the nodes (servers) of a cluster, on disk. Those nodes do the D&R parallel computation too. One critical outcome is that we can analyze a dataset whose total memory size is greater than the total memory of the cluster nodes. This is routine for us on in our data analysis projects, for projects of many others. There is much talk today about putting all data in memory to speed things up. We expect to add this to Tessera in the future, but it is a luxury, nice when feasible, but if relied on completely, is very limiting in the size and complexity of data that can be analyzed.

datadr

The datadr R package provides a high-level interface to D&R operations, making specification of divisions, analytic methods, and recombinations easy. One critical aspect is that it you can swap back ends without changing the user commands. Also, as new distributed computing technology comes along, datadr will be able to harness it. When the back end is Hadoop, datadr communicates with our RHIPE package coming below. datadr represents divisions as native R objects and allows D&R operations to be run against them. In addition to division and recombination methods, datadr also provides several indispensable tools for reading and manipulating data as well as a collection of standard division-independent statistical methods.

Trelliscope

Trelliscope is a visualization tool based on Trellis Display that enables scalable detailed visualization of data. Trellis Display operates by dividing the data into subsets. A visualization method is applied to each subset and shown on one panel of a multi-panel trellis display. This framework has proven to be powerful mechanism for all data, large and small. Trelliscope extends Trellis to big data. With big data, often there are too many subsets to look at all panels in a display. Trelliscope provides an interactive viewer that allows the analyst to sample, sort, and filter panels of a display on various quantities of interest.

RHIPE

RHIPE, the R and Hadoop Integrated Programming Environment, is a merger of R and Hadoop. It means ``in a moment'' in Greek, and is pronounced ``hree PAY.'' It provides communication between R and Hadoop. You can use it to carry out D&R data analysis. In R, the analyst passes the R code for division methods, analytic methods, and recombination methods to RHIPE R routines that communicate with Hadoop. In doing this, the analyst calls for certain Hadoop features such as key-value pairs, and the Map and Reduce computational operations. RHIPE does protect you from almost all of the immense detail of Hadoop. But datadr protects you even more. datadr communicates with RHIPE which then communicates with Hadoop. One way to think of this is that RHIPE entails progamming at a lower level than datadr.

Other Software in Tessera

Tessera makes use of software developed by others in addition to R and Hadoop. RHIPE uses Google protocol buffers to communicate with Hadoop, and it needs Oracle Java. There are two packages from RStudio. The first is RStudio Server, their Integrated Development Environment (IDE) for R, that for good reason is very widely used; it fits in nicely in the Tessera stack. The second is RStudio Shiny Server, used by Trelliscope to provide a web interface for the visualization.

Installation Overview

We provide four ways to get started with Tessera:

1 Workstation. 2 Vagrant Virtual Machine. 3 Amazon Elastic MapReduce. 4 Install on Your Own Cluster.

Just to review, here are the Tessera software components

  • R
  • datadr
  • Shiny Server
  • Hadoop
  • Trelliscope
  • Protocol Buffers
  • Rhipe
  • RStudio Server
  • Oracle Java

The first three are QuickStarts, very easy to carry out and described next. The second three have the full Tessera Software stack. For Vagrant and Amazon installation is quick and easy because the components are bundled up for you and ready to go. For Workstation it is quick and easy because you just install datadr, Trelliscope, and a couple of R packages.

Installing on your own cluster is more complex, and typically requires collaboration with your IT staff. Our cluster installation manual is here.

Install

Workstation Installation

After you have installed R, you can get going on a workstation simply by installing a few R packages. This setup will restrict you to data sizes commensurate with the hardware of your workstation, but it is sufficient to get a feel for Tessera and to analyze small or even moderate-sized data sets.

Once you are comfortable with the Tessera tools, you can plug into large-scale backends like Hadoop with virtually no changes to your code.

To install, simply launch R, install the devtools package from CRAN, and type the following: 

install.packages("devtools") # if not installed
library(devtools)
install_github("datadr", "tesseradata")
install_github("trelliscope", "tesseradata")

Vagrant Virtual Machine

To get a feel for running in a large-scale Tessera environment, we have provided a Vagrant setup that with a few simple commands allows you to provision a virtual machine with the entire Tessera software component running. The Vagrant script and instructions are available on Github.

Amazon Elastic MapReduce (EMR)

We have provided an easy way to get going with Tessera in a large-scale environment through a simple set of scripts that provision the Tessera environment on Amazon's Elastic MapReduce clusters. This allows you to spin up virtual clusters on-demand. An Amazon account is required.

This environment comes with RStudio Server running on the master node, so that all you need is a web browser to access R Studio, a fantastic R IDE that will be backed by your own Hadoop cluster.

The EMR scripts and instructions are available on Github.

Try It

Here is a simple example to get a feel for Tessera usage. For more compelling examples of Tessera in action, as well as in-depth tutorials, check out the Resources section. 

library(datadr); library(trelliscope)

# barley data as a local "distributed data frame"
barleyDdf <- ddf(barley)

# divide the ddf by the variable "site"
bySite <- divide(barleyDdf, by = "site")
bySite

# look at the first subset
bySite[[1]]

# look at the Grand Rapids subset
bySite[["site=Grand Rapids"]]

# get mean yield for each and recombine with "rbind"
recombine(bySite,
   apply = function(x) mean(x$yield),
   combine = combRbind())

## make a trelliscope display for the bySite division

# initialize visualization database (vdb) connection
vdbConn("vdb", autoYes = TRUE)

# make and test simple dotplot panel function
pf <- function(x)
   dotplot(variety ~ yield, data = x)
pf(bySite[[1]][[2]])

# make and test simple cognostics function
cf <- function(x) { list(
   meanYield = cogMean(x$yield, desc = "mean yield"),
   range = cogRange(x$yield, desc = "yield range"),
   min = cog(min(x$yield), desc = "min yield"),
   max = cog(max(x$yield), desc = "min yield")
)}
cf(bySite[[1]][[2]])

# add display with this panel and cog function to vdb
makeDisplay(bySite,
   name = "variety_vs_yield",
   desc = "test display with barley data",
   panelFn = pf, cogFn = cf,
   width = 400, height = 400,
   lims = list(x = "same", y = "free")
)

# view the display
view()
